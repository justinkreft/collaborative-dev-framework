# Collaborative Development Framework

**A project-agnostic framework for effective human-AI software collaboration**

## Purpose

This framework captures proven patterns, governance principles, and meta-learning strategies for building software collaboratively. It's designed to be applicable to any software project, regardless of technology stack, team size, or domain.

## What's Inside

### 🤝 [SOCIAL_CONTRACT.md](SOCIAL_CONTRACT.md)
Working relationship and communication principles:
- Communication principles (direct, conversational, options with reasoning)
- Working philosophy (build great, move fast/think carefully)
- Collaboration model (roles, challenge rights, disagreement handling)
- Working modes (exploration vs troubleshooting)
- Error recovery protocol
- Contract health checks

### 📋 [GOVERNANCE.md](GOVERNANCE.md)
Project governance framework covering:
- Core principles (transparency, quality, simplicity, collaboration, iteration)
- Decision-making processes
- Code quality standards
- Communication protocols
- Project lifecycle management
- Conflict resolution
- Templates for decisions, lessons, and development guides

### 🤝 [COLLABORATION_PATTERNS.md](COLLABORATION_PATTERNS.md)
Effective collaboration patterns for human-AI development:
- Work session patterns (exploration, direct implementation, iterative refinement)
- Communication best practices
- Decision-making frameworks
- Quality assurance approaches
- Problem-solving strategies
- Anti-patterns to avoid
- Session management templates

### 🧠 [META_LEARNING_FRAMEWORK.md](META_LEARNING_FRAMEWORK.md)
Learning how to learn - continuous improvement for software development:
- Reflection frameworks (After-Action Review, 5 Whys, Start-Stop-Continue)
- Learning capture patterns
- Feedback loop design (short, medium, long)
- Knowledge management strategies
- Continuous improvement cycles
- Learning taxonomies and levels
- Making learning stick

## How to Use This Framework

### For Solo Projects
- Use the principles to keep yourself accountable
- Document decisions for your future self
- Capture lessons learned to avoid repeating mistakes
- Focus on what adds value, drop what doesn't

### For Small Teams (2-5)
- Establish lightweight governance processes
- Regular retrospectives and check-ins
- Shared knowledge base
- Collaborative decision-making

### For Larger Teams (5+)
- Formal governance structures
- Comprehensive documentation
- Structured decision processes
- Knowledge sharing sessions

### For Open Source
- Public decision-making
- Contributor guidelines
- Transparent governance model
- Community involvement

## Adapting the Framework

**This framework is a starting point, not a rulebook.**

Take what works for your context:
- Simplify for fast-moving projects
- Expand for complex, long-term efforts
- Modify templates to match your style
- Drop practices that don't add value
- Add what's missing for your needs

## Origins

This framework emerged from collaborative development of real software projects, capturing what worked (and what didn't) in human-AI pair programming sessions. It distills lessons from:
- Bug fixes and reversions
- Architectural decisions and trade-offs
- Communication challenges and solutions
- Process experiments and refinements
- Meta-learning about how we learn

## Evolution

**This framework should evolve as you learn.**

Periodically ask:
- What's working?
- What's not being used?
- What's missing?
- What's changed in how we work?

Keep what works, drop what doesn't, add what's missing.
